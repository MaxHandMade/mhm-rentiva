# M2 — Database Bootstrap Contract: "Hybrid Reset"

**Context:** MHM Rentiva Test Harness
**Strategy:** Hybrid Reset
**Environment:** Windows 11 / XAMPP / Global WP Test Lib

## 0. The Problem
*   **Global Lib:** We rely on an external `wp-tests-config.php` which might point to a persistent development database.
*   **Permission Risk:** `DROP DATABASE` requires high privileges and might fail or be dangerous on shared local setups.
*   **State Leakage:** Custom tables and options created by the plugin might not be caught by WP Core's transaction rollback system.

## 1. The Strategy: "Hybrid Reset"

We will **NOT** attempt to drop/create the database itself.
We will **NOT** blindly truncate all tables (too risky for dev data).

Instead, we define two modes of operation:

### Mode A: Standard Run (`WP_TESTS_ISOLATE_DB=false`)
*   **Prefix:** Uses defined prefix in `wp-tests-config.php` (e.g., `wptests_`).
*   **Behavior:**
    *   **Core Data:** Relies 100% on `WP_UnitTestCase` transaction wrapping (start transaction -> run test -> rollback).
    *   **Plugin Data:**
        *   **Options:** Explicitly delete `mhm_rentiva_%` options in `tearDown()`.
        *   **Transients:** Explicitly delete `_transient_mhm_%` in `tearDown()`.
        *   **Custom Tables:** (None currently, but if added) `TRUNCATE` in `setUp/tearDown` within the test case.
*   **Safety Rule:** **NEVER DROP TABLES** in this mode.

### Mode B: Isolated Run (`WP_TESTS_ISOLATE_DB=true`)
*   **Prefix:** `wptests_{hash}_` (Generated by `bootstrap.php`).
*   **Behavior:**
    *   **Setup:** `bootstrap.php` standard installation creates fresh tables with this prefix.
    *   **Execution:** Tests run as normal.
    *   **Teardown (CRITICAL):** A `TestListener` monitors the suite. On `endTestSuite`, it executes `DROP TABLE IF EXISTS` for **ALL** tables matching the run's prefix.
*   **Safety Rule:** Only existing if the prefix strictly matches the generated hash format.

## 2. Implementation Specs

### 2.1 `bootstrap.php`
*   Must explicitly handle the `generated_prefix` logic.
*   Must expose the `generated_prefix` constant for the Listener to read.

### 2.2 `MHM_Test_Listener.php`
*   **Event:** `endTestSuite`
*   **Logic:**
    ```php
    if ( defined( 'MHM_TEST_ISOLATION_PREFIX' ) ) {
        $prefix = MHM_TEST_ISOLATION_PREFIX;
        // Query: SHOW TABLES LIKE '$prefix%'
        // Loop: DROP TABLE $table
    }
    ```

## 3. Benefits
1.  **Speed:** Standard runs don't incur install/drop overhead.
2.  **Safety:** Standard runs never destroy structure.
3.  **Correctness:** Isolated runs (CI/Nightly) guarantee zero state leakage.
